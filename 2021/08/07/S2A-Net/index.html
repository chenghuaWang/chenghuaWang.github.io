3.1 severe misalignment in one-stage detectors

  1. 以往的使用多个试探性的anchors的方法准确度在一些特定的情况下较低，例如： “桥” 这个实列，aspect ratio在1/30 左右，如果使用试探性的anchor，那么这几个anchors很有可能根本没有将桥包围在其中，或者只有很少的anchor将其包围在其中，因此，在之后的regress的过程中，acc会变得很低。同时，这种类似的情况下，大部分的anchors内部的实列的占比远小于外部的环境，这些由前后背景不平衡带来的影响，会有误差累积，会影响整个算法的performance.
  2. 标准的卷积操作得到的感受野是相对固定的，而且一般来说是axis-alignded。但是在遥感的图像中，各种实列分散并且有不同的角度，或者一些实列非常靠近、紧凑，且相同类别的实列也有不同的外观。因此，尽管anchor box对于某一个实列有着很高的置信度，卷积得到的特征和anchor box之间还是有很多的不对齐的。结果，分类的准确度很难反映出标定框的准确度，，这个也阻碍了之后post-processing阶段的表现。(e.g., non-maximum suppression(NMS))

为了解决这些单阶段遥感图像目标检测中普遍出现的问题。作者提出了一个新的算法，名为：Single-Shot Alignment Network $(S^{2}A-Net)$，这个算法包含了两个模块：1. Feature Alignment Module(FAM) 2. Oriented Detection Module(ODM)。其中 FAM 使用 Anchor Refinement Network(ARN) 来获得高质量的anchors， 然后使用Alignment Convolution(AlignConv)来自适应的让特征 anchors 去和GT的边界对齐。

4. proposed method

4.1 RetinaNet as Baseline

作者在文章中使用了一个单阶段的检测器 RetinaNet 来作为比较器。RetinaNet包含了1个 backbone network(特征金字塔，Feature pyramid network，FPN) 和 2个 task-specific subnetworks(fully convolutional networks with several stacked convolution layers)来进行分类和回归。并且使用Focal Loss来缓解前后背景样本不平衡带来的问题。

作者对RetinaNet最终回归的输出做了一些小改动，RetinaNet 最终会输出如下的oriented bounding box参数，其中$\theta$定义参见上图
